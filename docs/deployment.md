# Deployment

## Overview

planmd is deployed on Cloudflare Workers with a D1 (SQLite) database. The app is served at `planmd.dev` and `www.planmd.dev`.

## Infrastructure

| Resource | Details |
|---|---|
| **Worker** | `planmd` on Cloudflare Workers |
| **Database** | D1 `planmd-db` (ID: `58336148-65db-491e-a2d2-e53f77445164`) |
| **Domains** | `planmd.dev`, `www.planmd.dev` |
| **Auth** | Clerk custom domain at `clerk.planmd.dev` |
| **GitHub** | `mattbrandman/planmd` |
| **VCS** | Jujutsu (`jj`) with git backend |

## Manual Deploy

```bash
pnpm deploy   # builds + runs wrangler deploy
```

This runs `pnpm build` then `wrangler deploy`, which uploads the worker bundle and static assets.

## GitHub Integration (Workers Builds)

Auto-deploy on push to `main` via Cloudflare Workers Builds:

1. Dashboard → Workers & Pages → `planmd` → Settings → Builds → Connect to Git
2. Select `mattbrandman/planmd`
3. Build config:
   - **Build command:** `pnpm build`
   - **Deploy command:** `npx wrangler deploy`
   - **Branch:** `main`

## Database Migrations

Migrations are generated by Drizzle and live in `drizzle/`.

```bash
# Generate migration from schema changes
pnpm db:generate

# Apply to local D1
npx wrangler d1 migrations apply planmd-db --local

# Apply to production D1
npx wrangler d1 migrations apply planmd-db --remote
```

## Secrets

Production secrets are set via `wrangler secret put`:

```bash
npx wrangler secret put VITE_CLERK_PUBLISHABLE_KEY   # pk_live_xxx
npx wrangler secret put CLERK_SECRET_KEY              # sk_live_xxx
```

To list current secrets:

```bash
npx wrangler secret list
```

## Wrangler Config

`wrangler.jsonc` defines the worker configuration:

```jsonc
{
  "name": "planmd",
  "compatibility_date": "2025-09-02",
  "compatibility_flags": ["nodejs_compat"],
  "main": "@tanstack/react-start/server-entry",
  "routes": [
    { "pattern": "planmd.dev", "custom_domain": true },
    { "pattern": "www.planmd.dev", "custom_domain": true }
  ],
  "d1_databases": [
    {
      "binding": "planmd_db",
      "database_name": "planmd-db",
      "database_id": "58336148-65db-491e-a2d2-e53f77445164"
    }
  ]
}
```

## Version Control (Jujutsu)

This repo uses [Jujutsu (`jj`)](https://martinvonz.github.io/jj/) with a git backend. Common workflows:

```bash
# Make changes, describe, and push
jj describe -m "your change description"
jj bookmark set main -r @
jj git push --bookmark main

# After push, jj creates a new empty working copy on top
# This is normal — the previous change becomes immutable

# View log
jj log

# Create a new change on top of current
jj new
```

The GitHub remote is `origin` (`mattbrandman/planmd`). The `main` bookmark maps to the `main` branch on GitHub.
